# Security Policy

## 支持范围

当前安全响应范围覆盖主分支上的 Community Edition 代码。早期版本还没有长期维护分支，公开发布后会在这里补充受支持版本列表。

## 报告安全问题

请不要在公开 issue 中披露可被利用的安全问题。建议通过仓库维护者提供的私密渠道联系项目维护者；如果暂时没有公开安全邮箱，请先提交一个不包含细节的 issue，说明“希望私下报告安全问题”，维护者会跟进沟通方式。

报告时请尽量包含：

- 受影响版本或 commit。
- 可复现步骤。
- 影响范围。
- 是否涉及密钥、文件系统访问、任意代码执行、跨站脚本或用户素材泄露。

## 密钥和私有配置

不要提交以下内容：

- 云厂商 access key、secret、token。
- 私有 TTS 服务密钥。
- 证书、私钥、服务器路径。
- 线上域名反向代理配置。
- 商业二维码、充值码或后台配置。

开源版默认不需要私有后端。自定义 TTS Provider 应通过本地或自托管服务暴露安全的代理接口，不应在前端代码中硬编码云厂商密钥。

## 文件系统权限

项目使用 File System Access API 读写用户选择的本地项目文件夹。代码应只访问用户显式授权的目录，并避免将本地绝对路径写入可公开分享的项目文件、日志或文档。
