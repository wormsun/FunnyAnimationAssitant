# 沙雕动画小助手 表情管理系统 PRD (v3.1 Final)

## 1. 文档概述

### 1.1 项目背景
沙雕动画小助手 需要一个高效的表情管理模块，用于管理数字人的面部状态。表情分为“静止状态”和“说话状态”。为了解决素材对齐难、管理效率低的问题，本版本对原有设计进行了重构。

### 1.2 核心目标
1.  **扁平化管理**：放弃侧边栏，采用“画廊（Gallery）+ 标签筛选”模式。
2.  **所见即所得**：在统一弹窗中完成上传、对齐（锚点）、预览（按住说话）和属性配置。
3.  **精准对齐**：引入**锚点（Anchor Point）**功能，解决不同尺寸图片的对齐问题。
4.  **体验优化**：支持批量删除、创建时间倒序排列、增加图片容器边框。

## 2. 数据结构定义 (Schema)

```typescript
// 单帧图片资源
interface ExpressionFrame {
  id: string;          // UUID
  url: string;         // 图片地址 (Blob URL 或 CDN URL)
  file?: File;         // 原始文件对象 (仅前端上传时存在)
}

// 锚点坐标 (相对比例)
interface AnchorPoint {
  x: number;           // 0.0 - 1.0 (默认 0.5)
  y: number;           // 0.0 - 1.0 (默认 0.5)
}

// 表情实体对象
interface Expression {
  id: string;
  name: string;               // 表情名称，如 "开心"
  tags: string[];             // 标签分类，e.g. ["Male", "Happy"]
  
  // 核心资源
  defaultFrame: ExpressionFrame;      // 默认状态（必填）：不说话时的静止图
  speakingFrames: ExpressionFrame[];  // 说话状态（可选）：说话时的动画序列
  
  // 核心配置
  anchor: AnchorPoint;        // 锚点坐标，用于对齐面部中心
  speakingFps: number;        // 帧率，默认 30
  speakingLoop: boolean;      // 是否循环，默认 true
  
  // 元数据
  createdAt: number;          // 创建时间戳 (用于排序)
}
```

## 4. 功能需求详细说明

### 4.1 列表管理 (Gallery Features)

#### P0: 默认倒序排列
*   **规则**：列表加载时，严格按照 `createdAt` 降序排列。
*   **目的**：确保用户刚创建的表情直接显示在最前面。

#### P1: 视觉边框优化 (Visual Border)
*   **样式**：给所有表情图片容器（列表卡片、弹窗预览）增加 `1px solid #ddd` 边框。
*   **背景**：图片容器背景色设置为 **纯白色 (#FFFFFF)**。
*   **目的**：通过边框界定图片范围，保持界面整洁统一。

#### P1: 标签筛选 (Tab Filtering)
*   **交互**：顶部显示预设标签 Tab（如：全部、男性、女性、动物）。
*   **逻辑**：点击 Tab，前端过滤 `tags` 包含该值的表情。

### 4.2 核心编辑功能 (Editor Features)

#### P0: 锚点校准 (Anchor Point)
*   **可视化交互**：
    *   在左侧画布上显示一个红色的十字准星（⊕）。
    *   用户可鼠标拖拽该准星。
    *   拖拽时，右侧 `Anchor X/Y` 输入框数值实时更新。
*   **数据联动**：修改右侧数值，左侧准星位置也同步更新。
*   **默认值**：新建表情默认 `(0.5, 0.5)`（中心）。

#### P0: 智能导入 (Smart Import)
*   **场景**：用户新建表情时。
*   **逻辑**：若用户直接上传“说话动画序列”（多张图），系统自动提取动画序列的第一帧（按文件名排序后的第一张），**复制** 并填充到“默认图片”字段。

#### P0: 序列帧排序与管理
*   **排序**：上传多张图片后，系统根据**文件名**（如 `f_01.png`, `f_02.png`）自动进行一次排序。同时提供 `[🔃 按名称排序]` 手动按钮。

#### P0: 交互式预览 (Push-to-Talk)
*   **组件**：左侧底部的 `[ 🎤 按住说话测试 ]` 按钮。
*   **逻辑**：
    *   **Mouse Down (按住)**：预览窗口开始循环播放 `speakingFrames`。
    *   **Mouse Up (松开)**：预览窗口立即切换回 `defaultFrame`。

## 5. 开发实施建议

1.  **文件上传**：
    *   限制单张图片大小（推荐 2MB）。
    *   限制动画总帧数（推荐 60帧）。
2.  **图片加载**：
    *   列表页使用 `Lazy Load`。
    *   列表页仅加载压缩后的 `defaultFrame` 缩略图，不加载动画帧。
3.  **数据校验**：
    *   `Name` 必填。
    *   `DefaultFrame` 必填。
    *   `Anchor` 必须有值（默认0.5）。
4.  **UI细节**：
    *   卡片边框建议使用 `border-radius: 4px` 或 `8px`，增加圆角，视觉更柔和。
