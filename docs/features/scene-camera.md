# 相机系统完整功能 PRD (v6.2)

> **版本**: v6.2  
> **最后更新**: 2025-12-16  
> **核心变更**: 引入上帝视角编辑、显式动作类型切换、Follow/Shake 高级运镜支持。

---

## 1. 需求定义 (Requirements Definition)

### 1.1 核心概念
在 **Action Mode (导戏模式)** 下，相机系统不再仅仅是一个渲染视口，而是一套完整的**可视化运镜工具**。

*   **上帝视角 (God View)**：编辑时，画布世界保持静止，用户通过操作一个**“绿色相机框对象”**来定义取景范围。
*   **所见即所得 (WYSIWYG)**：相机框的每一次拖拽、缩放、旋转，都会实时映射为数据并更新预览。
*   **槽位驱动 (Slot-Driven)**：所有的相机动作（运镜）都严格依附于 TTS 分句（Slot）。

### 1.2 功能列表

| 功能模块 | 需求描述 | 优先级 |
| :--- | :--- | :--- |
| **相机可视化** | 在画布上绘制 16:9 绿色线框。 | P0 |
| **显式模式切换** | 属性面板提供 Tab 切换：`切(Cut)` / `运(Move)` / `跟(Follow)` / `震(Shake)`。 | P0 |
| **镜头切 (Cut)** | 瞬时动作。拖拽相机框直接设定当前槽位起始的静态机位。 | P0 |
| **运镜 (Move)** | 持续动作。拖拽相机框设定当前槽位结束时的目标机位，系统自动补间。 | P0 |
| **跟随 (Follow)** | 持续动作。将相机中心锁定在指定 场景对象 身上，支持阻尼和偏移。 | P1 |
| **震动 (Shake)** | 持续动作。参数化控制画面的随机抖动（强度、频率、衰减）。 | P1 |

## 2. UI/UX 设计 (User Interface)

### 2.1 画布交互设计 (Canvas Interaction)

**A. 未选中相机时**
*   **显示**：仅显示绿色的矩形线框，作为构图参考。

**B. 选中相机时 (编辑状态)**
*   **显示**：
    *   **绿色矩形线框** 。
    *   **控制手柄**：四角缩放手柄 (Zoom)、中心拖拽区域 (Pan)。
*   **交互**：
    *   **拖拽中心**：改变 `x`, `y`。
    *   **拖拽四角**：改变 `zoom` (反向逻辑：框变大=Zoom Out，框变小=Zoom In)。

### 2.2 属性面板设计 (ActionInspector)
