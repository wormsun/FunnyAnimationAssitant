# 沙雕动画小助手 - 动作编排器 (Action Sequencer) PRD v6.2

**文档状态**：**Final** (定稿版)
**所属模块**：场景编辑器 - 导戏模式 (Action Mode)
**UI 位置**：底部侧边栏 (Bottom Panel)
**核心理念**：文本即标尺，属性即动作。
**最后更新**：2025-12-13

---

## 1. 概述 (Overview)

### 1.1 设计目标
在 v6.2 架构中，动作编排器不再是一个“时间轴编辑器”，而是一个 **“叙事节奏编排器”**。它解耦了动作与绝对时间的强绑定，让用户专注于“这句话讲到哪里时发生什么动作”。

### 1.2 核心设计哲学
1.  **文本节奏流 (Text Rhythm Stream)**：彻底摒弃秒数显示。槽位卡片的宽度由**文本字数**决定。用户感知的不是“2.5秒”，而是“一个长句子的跨度”。
2.  **对象即轨道 (Object as Track)**：每个场景对象拥有独立轨道，解决并发动作的可视化问题。
3.  **属性驱动创建 (Property-Driven Creation)**：**Timeline 仅用于调整**，所有动作的创建均由 **Inspector (属性面板)** 的状态修改或触发按钮驱动。
4.  **极简主义 (Minimalism)**：移除播放控制、吸附开关、可见性开关等冗余元素，降低认知负担。

## 2. 界面布局 (UI Layout)

组件位于界面底部，采用 **“左侧对象头 + 右侧节奏流”** 的布局。

### 2.1 容器框架

```text
+-------------------------------------------------------------------------------------------+
|  [=== 顶部拖拽手柄 (Resizer Handle) ===]                                   [▼ 折叠]       |
+-------------------------------------------------------------------------------------------+
|  [工具栏 Toolbar]                                                                         |
|  过滤器: [👁️ 仅活跃对象 v]   |   🔍 缩放: [-]----O----[+]                                |
+-----------------------+-------------------------------------------------------------------+
|  (A) 左侧对象头       |  (B) 右侧轨道区域 (水平滚动, 垂直滚动)                            |
|  (Fixed Width: 160px) |                                                                   |
|                       |  +-- #1 --+  +------- #2 -------+  +-------------- #3 ----------+ |
|  [ 🗣️ 参考台词 ]      |  | "嗨！" |  | "今天天气不错"   |  | "我们要不要去公园玩？"   | |
|  (不可点击)           |  +--------+  +------------------+  +--------------------------+ |
|                       |                                                                   |
| --------------------- | ----------------------------------------------------------------- |
| [ 🎥 相机 ]            | [========== 🟩 推近 ======>]                                       |
| --------------------- | ----------------------------------------------------------------- |
| [ 👦 小明 ]            | [🟦 移..]   [====== 🟦 移动: 走到路中间 (Span:1) ======>]           |
|                       | [🙂]                                                               |
| --------------------- | ----------------------------------------------------------------- |
| [ 🎵 开门声 ]          | [🔊]                                                               |
+-----------------------+-------------------------------------------------------------------+
```

### 2.2 区域详细定义

#### (A) 左侧对象头 (Track Headers)
*   **宽度**：固定 160px。
*   **内容**：`[图标] + [对象名称]`。
*   **交互**：
    *   **点击**：**选中该对象**。
        *   *功能*：这是解决画布遮挡问题的核心入口。点击后，属性面板切换至该对象，画布显示该对象的选中框。
        *   *视觉反馈*：背景高亮。
    *   **右键**：显示“删除该对象的所有动作”选项。

#### (B) 右侧轨道区域 (Timeline Tracks)
*   **第一行：台词节奏流 (Subtitle Rhythm Stream)**
    *   **性质**：全局“时间”标尺，始终吸附顶部。
    *   **卡片内容**：仅显示 `Slot Index` (#1) 和 `Text` ("今天天气不错")。**不显示时长**。
    *   **交互**：点击卡片 -> 切换 `currentSlotIndex`，画布重绘至该分句起始状态。
*   **后续行：对象动作轨 (Object Tracks)**
    *   **网格背景**：垂直虚线严格对齐上方 Slot 的边界。
    *   **内容**：根据 `Action.target` 渲染对应的 UI 组件。

#### (C) 工具栏 (Toolbar)
*   **过滤器 (Filter)**：
    *   `👁️ 仅活跃对象` (默认)：仅显示当前 Block 内有动作的对象 + 相机。
    *   `📂 全部显示`：显示所有对象。
    *   `✨ 仅选中`：只显示当前选中的对象。
    *   `⚙️ 自定义...`：弹出选择面板（见下文）。
*   **缩放 (Zoom)**：调整字数与像素宽度的比例系数。

## 3. 核心功能与逻辑 (Core Logic)

### 3.1 节奏流渲染逻辑 (Rhythm Rendering)
我们彻底摒弃了物理时间计算，采用**字数权重**来决定卡片宽度。

*   **宽度公式**：
    `CardWidth = BasePadding + (CharacterCount * ZoomFactor)`
    *   `BasePadding`: 基础宽度 (如 40px)，保证极短的词也有点击区域。
    *   `ZoomFactor`: 缩放系数 (如 15px/字)。
*   **优势**：
    *   **极其稳定**：无论 TTS 是否生成、语速如何变化，界面布局永远稳固，不会跳动。
    *   **所见即所得**：长句子的格子自然就长，符合直觉。
*   **TTS 状态**：无需任何 Loading 状态或 Pending 样式。对于编辑器而言，**文本确定，宽度即确定**。

### 3.2 动作可视化
根据 `Action.category` 渲染：

| 类型                | UI 形态         | 颜色编码               | 布局规则                               |
| :------------------ | :-------------- | :--------------------- | :------------------------------------- |
| **瞬时 (Point)**    | **图标 (Icon)** | 灰底/彩色图标          | 居中显示。同一格多个图标自动水平排列。 |
| **持续 (Duration)** | **长条 (Bar)**  | 变换:🟦, 相机:🟩, 特效:🟪 | 横跨 `slotSpan` 个网格，高度 60%。     |

*   **图标示例**：表情(🙂), 姿态(🧍), 音效(🔊), 特效(✨), 显隐(👁️)。

### 3.3 自定义筛选器 (Custom Filter)
*   **入口**：过滤器下拉 -> `自定义...`。
*   **交互**：弹出 Popover 面板。
    *   提供搜索框。
    *   按类型（角色/道具/特效/声音）折叠展示列表。
    *   用户勾选需要的轨道，点击外部关闭即应用。

## 4. 交互工作流 (Workflow)

### 4.1 核心设计：操作入口决定动作类型

**Action Mode = 始终处于录制状态**，无需手动开启/关闭录制模式。

用户的操作入口自动决定生成的动作类型：

| 操作入口                 | 操作方式                      | 生成的动作类型                 | 动作分类 |
| ------------------------ | ----------------------------- | ------------------------------ | -------- |
| **画布 (Canvas)**        | 拖拽对象移动位置              | `tween_transform`              | Duration |
| **画布 (Canvas)**        | 拖拽缩放/旋转                 | `tween_transform`              | Duration |
| **属性面板 (Inspector)** | 修改 x/y/scale/rotation/alpha | `set_transform`                | Point    |
| **属性面板 (Inspector)** | 修改 pose/expression          | `set_character`                | Point    |
| **属性面板 (Inspector)** | 修改 visible                  | `set_active`                   | Point    |
| **属性面板 (Inspector)** | 点击触发按钮 (震动/音效)      | `camera_shake` / `vfx_trigger` | Point    |

### 4.2 动作创建流程

**Timeline 不再提供任何创建入口（无右键菜单、无加号）。**

#### 场景 1：创建表情/姿态变化 (Point Action)
```
1. Timeline 点击 Slot #2（选择时刻）
2. Timeline 左侧点击"小明"（选择对象）
3. Inspector 修改表情下拉框（修改属性）
4. -> 系统自动在 Slot #2 生成 set_character 动作
5. -> Timeline 显示 🙂 图标
```

#### 场景 2：创建移动动画 (Duration Action)
```
1. Timeline 点击 Slot #2（选择时刻）
2. Canvas 直接拖拽小明到新位置（画布操作）
3. -> 系统自动在 Slot #2 生成 tween_transform 动作
4. -> Timeline 显示 🟦 长条
```

#### 场景 3：创建瞬移 (Point Action)
```
1. Timeline 点击 Slot #2（选择时刻）
2. 选中小明，在 Inspector 修改 X/Y 坐标
3. -> 系统自动在 Slot #2 生成 set_transform 动作
4. -> Timeline 显示 ◆ 图标
```

### 4.3 同槽位同对象的动作合并策略

当用户在同一槽位对同一对象进行多次属性修改时：

| 情况         | 处理策略       | 示例                                                              |
| ------------ | -------------- | ----------------------------------------------------------------- |
| **相同属性** | 更新现有动作   | 先设 x=100，再设 x=150 → 最终 `{ x: 150 }`                        |
| **不同属性** | 合并到同一动作 | 先设 x=100，再设 y=200 → 最终 `{ x: 100, y: 200 }`                |
| **不同类型** | 分别创建动作   | 修改 x 创建 `set_transform`，修改 expression 创建 `set_character` |

**合并逻辑示例**：
```typescript
// 用户操作序列：
// 1. 修改 x = 100  →  set_transform { params: { x: 100 } }
// 2. 修改 y = 200  →  set_transform { params: { x: 100, y: 200 } }  // 合并
// 3. 修改 x = 150  →  set_transform { params: { x: 150, y: 200 } }  // 更新
// 4. 修改 expression = 'smile' → 新增 set_character { params: { expression: 'smile' } }
```

### 4.4 动作调整 (Adjustment)
**Timeline 专注于时序调整。**

*   **吸附 (Snapping)**：默认强制开启。
    *   拖拽动作时，自动吸附网格左边缘。
    *   调整时长时，自动吸附网格分界线。
*   **移动**：拖拽 Icon/Bar 到其他 Slot。
*   **改变跨度**：拖拽 Bar 右边缘，改变 `slotSpan`。
*   **删除**：选中 -> `Delete` 键。

## 5. 数据处理建议 (Data Handling)

### 5.1 数据转换 (ViewModel)
前端需要一个 Computed 属性将数据转为轨道格式：

```typescript
interface RhythmTimelineData {
  slots: {
    index: number;
    text: string;
    pixelWidth: number; // 基于字数计算
  }[];
  
  tracks: {
    objectId: string;
    actions: {
      type: 'icon' | 'bar';
      slotIndex: number;
      slotSpan: number; // For bar only
      display: any; // Icon char or Label text
    }[];
  }[];
}
```

### 5.2 状态 vs 触发的底层处理
虽然 UI 上不区分，但底层求值器 (`ActionEvaluator`) 需区分：
*   **Trigger (音效/震动)**：`isJustEnteredSlot` 时执行一次。
*   **State (表情/位置)**：`findLastState` 回溯并应用。
